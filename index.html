02 — 노션 세팅
CSV import 3단계
📥 pikk-storyboard-notion.csv → 노션 DB
1
노션 페이지에서 새 DB 생성
산출물 보드 페이지 하위에 "스토리보드 현황" DB 생성.
/database 입력 → Full page database 선택.
2
CSV 가져오기
DB 우상단 ··· → Merge with CSV → pikk-storyboard-notion.csv 업로드.
첨부 파일: pikk-storyboard-notion.csv
3
속성 타입 조정 (import 후)
노션은 모든 속성을 Text로 import함. 아래 속성만 타입 변경 필요.
속성명변경 타입이유
담당자SelectA/B 필터링
플로우Select온보딩/픽/응모/마이 필터
상태Status작성중→검토요청→확정 워크플로
플랫폼Multi-select앱전용/웹공통 복수 선택
PhaseSelect1/1.5/2 필터
스펙 링크URLHTML 뷰어 링크 연결
피그마 링크URL피그마 프레임 직링크
✅
import 완료 후 "담당자: A" 필터 뷰와 "담당자: B" 필터 뷰 2개를 저장해두면 각자 자기 화면만 볼 수 있습니다.
03 — HTML 뷰어 연결
스펙 링크 채우기
🔗 노션 DB ↔ HTML 뷰어 연결 방법
A
GitHub Pages로 호스팅 (권장)
pikk-storyboard-filled.html → index.html로 이름 변경 후 GitHub 레포에 업로드 → Pages 활성화.
https://{username}.github.io/pikk-storyboard/
B
노션 DB "스펙 링크" 속성에 URL 입력
모든 화면의 스펙 링크 = 동일한 HTML 뷰어 URL.
(뷰어 자체에서 화면을 선택해서 보는 구조이므로 URL 1개로 통일 가능)
스펙 링크: https://{username}.github.io/pikk-storyboard/
C
노션 페이지에 뷰어 임베드 (선택)
산출물 보드 페이지에서 /embed → URL 입력 → 노션 안에서 직접 열람 가능.

05 — 협업 규칙
이것만 지키면 충돌 없습니다
규칙내용이유
스펙 수정 HTML 파일만 수정. 노션 페이지 본문에 스펙 내용 적지 않기. 두 곳을 동기화하면 반드시 불일치 발생
상태 변경 노션 DB에서만 상태 변경. 검토요청 시 슬랙으로 상대방 멘션. 알림 없이 상태 바꾸면 누락됨
피드백 노션 DB 해당 페이지 댓글에 작성. 반영 시 댓글에 ✅ 체크. 슬랙 피드백은 묻힘
확정 조건 A·B 둘 다 댓글 이슈 0개 + 피그마 링크 입력 완료. 한쪽만 확정하면 의미 없음
HTML 업데이트 스펙 내용 바뀌면 노션 DB 메모란에 "v1.1 업데이트" 기록. 어떤 버전을 보고 있는지 추적
